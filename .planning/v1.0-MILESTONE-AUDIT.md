## Critical Findings

### Finding 1 (High): Raw eval path produces zero output

`eval_raw()` in `crates/cintx-compat/src/raw.rs` calls `evaluate()` but discards the executor's staging output. A fresh zero-filled vector is created and written to the caller's output buffer. Affects: `cintrs_eval`, C ABI callers, oracle comparison.

The safe facade path (`SessionQuery::evaluate()`) is **not** affected — it uses `RecordingExecutor` which captures `io.staging_output()`.

**Affected requirements:** COMP-01, COMP-04, EXEC-02, EXEC-04, EXEC-05, VERI-01

### Finding 2 (Medium): Capability fingerprint drift detection inoperative

`bootstrap_wgpu_runtime()` computes a real fingerprint but `preflight_wgpu()` discards it. The `BackendCapabilityToken.capability_fingerprint` stays at 0 throughout execution. D-08 drift detection for capability changes is mechanically inoperative.

**Affected requirements:** COMP-05
