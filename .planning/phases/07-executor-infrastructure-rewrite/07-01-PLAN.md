Define:
```rust
pub mod wgpu_backend;
pub mod cpu_backend;

use std::collections::HashMap;
use std::sync::Mutex;
use cintx_core::cintxRsError;
use cintx_runtime::{BackendIntent, BackendKind};

pub enum ResolvedBackend {
    Wgpu(cubecl_runtime::client::ComputeClient<cubecl_wgpu::WgpuRuntime>),
    #[cfg(feature = "cpu")]
    Cpu(cubecl_runtime::client::ComputeClient<cubecl::cpu::CpuRuntime>),
}
```

Note: The Cpu arm is behind `#[cfg(feature = "cpu")]` because `cubecl::cpu::CpuRuntime` only exists when the `cpu` feature is enabled.

Add `ResolvedBackend::from_intent(intent: &BackendIntent) -> Result<Self, cintxRsError>`:
- Match on `intent.backend`:
  - `BackendKind::Wgpu` -> call `wgpu_backend::resolve_wgpu_client(intent)` which uses the existing `bootstrap_wgpu_runtime` for preflight, then calls `cubecl_wgpu::WgpuRuntime::client(&device)` to get the ComputeClient. The wgpu device is derived from `intent.selector` using the same selector parsing as `runtime_bootstrap.rs`.
  - `BackendKind::Cpu` -> call `cpu_backend::resolve_cpu_client()` which calls `cubecl::cpu::CpuRuntime::client(&cubecl::cpu::CpuDevice::default())`.
  - When `BackendKind::Cpu` is requested but `cpu` feature is not enabled, return `UnsupportedApi { requested: "cpu-backend:feature-not-enabled" }`.

Add `resolve_backend_kind() -> BackendKind` function (per D-04, D-10):
- Read `std::env::var("CINTX_BACKEND")`:
  - `Ok("cpu")` -> `BackendKind::Cpu`
  - `Ok("wgpu")` | `Err(_)` -> `BackendKind::Wgpu` (default to wgpu when unset per D-10)
  - `Ok(other)` -> `tracing::warn!` and default to `BackendKind::Wgpu`

Add a `BackendCache` struct:
```rust
pub(crate) struct BackendCache {
    cache: Mutex<HashMap<String, ResolvedBackend>>,
}
```
The cache key is `format!("{}:{}", backend_kind_str, selector)`.
`BackendCache::resolve(&self, intent: &BackendIntent) -> Result<&ResolvedBackend, cintxRsError>` — checks cache first, creates on miss.

IMPORTANT: `ResolvedBackend` must implement `Clone` if `ComputeClient` is `Clone`. If `ComputeClient` is not `Clone`, store `Arc<ResolvedBackend>` in the cache instead.

**Create `crates/cintx-cubecl/src/backend/wgpu_backend.rs`:**
```rust
pub fn resolve_wgpu_client(intent: &BackendIntent) -> Result<ComputeClient<WgpuRuntime>, cintxRsError> {
    // 1. Run preflight via bootstrap_wgpu_runtime to validate capability
    let report = crate::runtime_bootstrap::bootstrap_wgpu_runtime(intent)?;
    if !report.is_capable() {
        return Err(capability_error(report));
    }
    // 2. Parse selector to WgpuDevice
    let device = selector_to_device(&intent.selector)?;
    // 3. Return client
    Ok(cubecl_wgpu::WgpuRuntime::client(&device))
}
```
Reuse `selector_to_wgpu_device` logic from `runtime_bootstrap.rs` (either re-export or duplicate the small match). The `bootstrap_wgpu_runtime` call handles the `OnceLock` guard to prevent double-init panics.
