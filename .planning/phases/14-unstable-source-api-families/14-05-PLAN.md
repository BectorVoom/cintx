**1. Extend profile validation in xtask/src/oracle_update.rs:**

The `validate_required_profile_scope` function currently hard-rejects any profile not in PHASE4_APPROVED_PROFILES (the standard 4). The nightly CI job needs to pass `--profiles "unstable-source"` which will be rejected.

**Solution:** Create a separate validation path for the unstable-source profile. Two approaches:

**Approach A (preferred):** Add `"unstable-source"` as a known profile in `ensure_known_profile`. Then modify `validate_required_profile_scope` to ONLY validate that the 4 required profiles are present when the requested set is exactly the standard 4 — but when the requested set is `["unstable-source"]` alone, accept it without requiring the standard 4. This allows two valid invocations:
- `--profiles "base,with-f12,with-4c1e,with-f12+with-4c1e"` — standard gate
- `--profiles "unstable-source"` — nightly unstable gate

Implementation:
```rust
const ALL_KNOWN_PROFILES: &[&str] = &[
    "base", "with-f12", "with-4c1e", "with-f12+with-4c1e", "unstable-source"
];

fn validate_required_profile_scope(profiles: &[String]) -> Result<Vec<String>> {
    // Validate all requested profiles are known
    for p in profiles {
        if !ALL_KNOWN_PROFILES.contains(&p.as_str()) {
            bail!("unknown profile `{p}`, expected one of: {}", ALL_KNOWN_PROFILES.join(", "));
        }
    }
    // If requesting unstable-source, it runs standalone (per D-02)
    if profiles.iter().any(|p| p == "unstable-source") {
        if profiles.len() != 1 || profiles[0] != "unstable-source" {
            bail!("unstable-source profile must be run standalone, not combined with other profiles");
        }
        return Ok(vec!["unstable-source".to_owned()]);
    }
    // Standard 4-profile validation
    let requested: BTreeSet<String> = profiles.iter().cloned().collect();
    let required: BTreeSet<String> = PHASE4_APPROVED_PROFILES.iter().map(|p| (*p).to_owned()).collect();
    let missing: Vec<String> = required.difference(&requested).cloned().collect();
    let extra: Vec<String> = requested.difference(&required).cloned().collect();
    if !missing.is_empty() || !extra.is_empty() {
        bail!("profile scope mismatch, expected exactly `{REQUIRED_PROFILE_CSV}` (missing: {:?}, extra: {:?})", missing, extra);
    }
    Ok(PHASE4_APPROVED_PROFILES.iter().map(|p| (*p).to_owned()).collect())
}

fn ensure_known_profile(profile: &str) -> Result<()> {
    if ALL_KNOWN_PROFILES.contains(&profile) { return Ok(()); }
    bail!("unsupported profile `{profile}`, expected one of: {}", ALL_KNOWN_PROFILES.join(", "))
}
```

**Job definition:**

```yaml
unstable_source_oracle:
    name: "unstable_source_oracle (nightly)"
    if: github.event_name == 'schedule' || github.event_name == 'workflow_dispatch'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Resolve Rust toolchain
        id: toolchain
        run: |
          CHANNEL=$(grep -oP 'channel\s*=\s*"\K[^"]+' rust-toolchain.toml)
          echo "channel=$CHANNEL" >> "$GITHUB_OUTPUT"

      - name: Install Rust
        uses: dtolnay/rust-toolchain@master
        with:
          toolchain: ${{ steps.toolchain.outputs.channel }}

      - name: Cache cargo
        uses: actions/cache@v4
        with:
          path: |
            ~/.cargo/registry
            ~/.cargo/git
            target
          key: ${{ runner.os }}-cargo-unstable-${{ hashFiles('**/Cargo.lock') }}

      - name: Run unstable-source oracle parity
        env:
          CINTX_ORACLE_BUILD_VENDOR: "1"
        run: |
          cargo test -p cintx-oracle --features cpu,unstable-source-api -- unstable_source_parity --test-threads=1

      - name: Run xtask unstable-source oracle-compare
        env:
          CINTX_ORACLE_BUILD_VENDOR: "1"
        run: |
          cargo run --manifest-path xtask/Cargo.toml -- oracle-compare \
            --profiles "unstable-source" \
            --include-unstable-source true
```

**Trigger condition:** The job ONLY runs on `schedule` or `workflow_dispatch` events (D-11 nightly gate). It does NOT run on `push` or `pull_request` events.
