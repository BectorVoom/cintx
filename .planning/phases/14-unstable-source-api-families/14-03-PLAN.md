**2. Implement launch_grids in unstable.rs:**

Replace the stub `launch_grids` with a real implementation. The grids kernel:

a. Reads `grids_params` from `plan.operator_env_params.grids_params`. If None, return `InvalidEnvParam` (validator should have caught this, but defense in depth).

b. Extracts `ngrids` and `ptr_grids` from GridsEnvParams.

c. Per D-06, output shape is `(ncomp * ngrids * di * dj)`. The output buffer must be sized accordingly.

d. For each grid point `g` in 0..ngrids:
   - Read grid coordinates from env at `ptr_grids + g*3`: `grid_xyz = [env[ptr_grids + g*3], env[ptr_grids + g*3 + 1], env[ptr_grids + g*3 + 2]]`
   - Compute the 1e integral using the grid point as the "nuclear" center displacement: the center displacement vector becomes `Ri - grid_xyz` instead of `Ri - Rj`
   - Write output at offset `g * ncomp * di * dj` within the output buffer

e. Per libcint `g1e_grids.c`: the grids integral uses the same Rys quadrature as standard 1e overlap, but with the center at the grid point instead of the nuclear center. The `CINTg0_1e_grids` function modifies the center displacement: `ri[xyz] = Ri[xyz] - grids[grid_id*3 + xyz]`.

f. Match on `operator_name` for variant dispatch:
   - `"grids"` — base int1e_grids, ncomp=1
   - `"grids_ip"` — ip derivative, ncomp=3
   - `"grids_ipvip"` — ipvip derivative, ncomp=9
   - `"grids_spvsp"` — spvsp, ncomp=4
   - `"grids_ipip"` — second derivative, ncomp=9

g. Each variant has its own ng array (from cint_funcs.h / cint1e_grids.c). Read the libcint source to determine the exact ng values and gout computation for each variant.

h. The grids kernel uses `CINT1e_drv` with `c2s_sph_1e` cart-to-sph transform, same as standard 1e. The only difference is the grid-displaced center in the G-tensor evaluation.

fn build_h2o_sto3g_grids(ngrids: usize) -> (Vec<i32>, Vec<i32>, Vec<f64>, Vec<i32>) {
    let (atm, bas, mut env) = build_h2o_sto3g();
    env[11] = ngrids as f64;           // env[NGRIDS] = count

}
