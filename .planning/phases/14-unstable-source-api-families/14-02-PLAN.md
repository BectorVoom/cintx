**launch_origi implementation:**

Origi kernels are 1e integrals with origin-displaced r^n operators. The G1E_R_I macro computes `g[i+1] = g[i] * (ri - R_origin)` where R_origin is env[PTR_COMMON_ORIG] (env[1..4], zero by default).

Per research ng arrays:
- `int1e_r2_origi`: ng=[2,0,0,0,2,1,1,1], ncomp=1. gout = sum over xyz of g3[ix]*g0[iy]*g0[iz] (r^2 = x^2+y^2+z^2)
- `int1e_r4_origi`: ng=[4,0,0,0,4,1,1,1], ncomp=1. gout = r^4 polynomial
- `int1e_r2_origi_ip2`: ng=[2,1,0,0,3,1,1,3], ncomp=3. gradient of r^2 on ket
- `int1e_r4_origi_ip2`: ng=[4,1,0,0,5,1,1,3], ncomp=3. gradient of r^4 on ket

Follow the pattern in `one_electron.rs` — use `launch_one_electron` infrastructure with variant-specific gout dispatch. The `operator_name` from the manifest entry (`r2_origi`, `r4_origi`, `r2_origi_ip2`, `r4_origi_ip2`) determines which gout to use.

Match on `plan.descriptor.entry.operator_name` to select the gout variant, then call into the 1e kernel infrastructure. The G1E_R_I steps use the existing VRR machinery — the recurrence adds angular momentum on the i-index relative to the origin displacement vector.

**launch_origk implementation:**

Origk kernels are 3c1e integrals with r^n operator on the k-shell (third center). Per research ng arrays:
- `int3c1e_r2_origk`: ng=[0,0,2,0,2,1,1,1], ncomp=1
- `int3c1e_r4_origk`: ng=[0,0,4,0,4,1,1,1], ncomp=1
- `int3c1e_r6_origk`: ng=[0,0,6,0,6,1,1,1], ncomp=1
- `int3c1e_ip1_r2_origk`: ng=[1,0,2,0,3,1,1,3], ncomp=3
- `int3c1e_ip1_r4_origk`: ng=[1,0,4,0,5,1,1,3], ncomp=3
- `int3c1e_ip1_r6_origk`: ng=[1,0,6,0,7,1,1,3], ncomp=3

Follow `center_3c1e.rs` pattern. The G1E_R_K macro applies the same recurrence but on the k-index. Match on operator_name to select gout variant.

**launch_ssc implementation:**

SSC uses the same gout as standard `int3c2e_sph` (`CINTgout2e`) but with a different cart-to-sph transform (`c2s_sph_3c2e1_ssc` instead of `c2s_sph_3c2e1`). The SSC c2s applies different normalization. Per research: ng=[0,0,0,0,0,1,1,1], identical to standard int3c2e.

Follow `center_3c2e.rs` — route through the same G-tensor and gout computation, but apply the SSC c2s variant in the transform step. The `is_ssc` flag controls the normalization factors.

Read the libcint C source carefully for each gout function to get the exact angular momentum iteration patterns and accumulation formulas. Port them faithfully to Rust host-side implementations following the `*_host()` convention from Phase 8.
