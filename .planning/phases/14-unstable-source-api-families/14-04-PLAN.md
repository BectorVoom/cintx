Replace the stub `launch_breit` with a real implementation following D-08 (single composite kernel, single output buffer).

**Breit composition algorithm (per breit.c _int2e_breit_drv):**

The Breit interaction is decomposed as: `Breit = -0.5 * (Gaunt + gauge_r1) + 0.5 * gauge_r2`

Equivalently per the C code:
1. Compute Gaunt integral -> `buf1`
2. Compute gauge_r1 -> `buf`; then `buf1[i] = -buf1[i] - buf[i]`  (negate Gaunt, subtract gauge_r1)
3. Compute gauge_r2 -> `buf`; then `buf1[i] = (buf1[i] + buf[i]) * 0.5`  (add gauge_r2, halve)
4. Copy `buf1` -> output

**For int2e_breit_r1p2_spinor:**
- Gaunt sub-kernel: standard 2e spinor with Gaunt gout (CINTgout2e_int2e_r1p2 with ng={0,2,0,1,2,1,1,1})
- gauge_r1 sub-kernel: CINTgout2e_int2e_gauge_r1_r1p2 with ng={2,2,0,1,4,1,1,1}
- gauge_r2 sub-kernel: CINTgout2e_int2e_gauge_r2_r1p2 with ng={2,0,0,1,2,1,1,1}

**For int2e_breit_r2p2_spinor:**
- Gaunt sub-kernel: CINTgout2e_int2e_r2p2 with ng={0,2,0,1,2,1,1,1}
- gauge_r1 sub-kernel: CINTgout2e_int2e_gauge_r1_r2p2 with ng={2,2,0,1,4,1,1,1}
- gauge_r2 sub-kernel: CINTgout2e_int2e_gauge_r2_r2p2 with ng={2,0,0,1,2,1,1,1}

**Implementation approach:**

1. Match on `plan.descriptor.entry.operator_name`: `"breit_r1p2"` or `"breit_r2p2"`

2. For spinor output: the output size is complex-valued (interleaved re/im). Size: `nop = ncomp * di_spinor * dj_spinor * dk_spinor * dl_spinor * 2` (factor 2 for complex).

3. Allocate two host-side buffers: `buf1: Vec<f64>` (size nop) and `buf: Vec<f64>` (size nop). Per Pitfall 6: do NOT alias with the output buffer.

4. Compute Gaunt sub-kernel into `buf1` using 2e spinor infrastructure with the Gaunt gout function.

5. Compute gauge_r1 sub-kernel into `buf` using 2e spinor infrastructure with gauge_r1 gout function.

6. Accumulate: `for i in 0..nop { buf1[i] = -buf1[i] - buf[i]; }`

7. Compute gauge_r2 sub-kernel into `buf` using 2e spinor infrastructure with gauge_r2 gout function.

8. Accumulate: `for i in 0..nop { buf1[i] = (buf1[i] + buf[i]) * 0.5; }`

9. Copy `buf1` into `output`.

10. Return `ExecutionStats` with combined metrics from all three sub-kernel launches.

**Gout functions:** Read the exact gout implementations from `breit.c` and `g2e.c`. Each gout uses combinations of G2E_D_L (angular momentum derivative on l), G2E_R0J (position relative to center j), G2E_D_J (derivative on j), G2E_D_I (derivative on i). Port these as Rust host-side functions.

**Representation guard:** The manifest entries for Breit have `forms: &["spinor"]` only (D-07). The `resolve_family` in `mod.rs` already checks `supports_representation` which will reject cart/sph. The `launch_breit` function can add a defensive check: if representation is not Spinor, return `UnsupportedApi { requested: "breit requires spinor representation" }`.
