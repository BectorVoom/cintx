**Step 1 — Fix spinor-first validation (D-05, 4C1E-03):**

In `ensure_validated_4c1e`, reorder checks so spinor representation is checked FIRST, before the feature gate or any other validation:

```rust
fn ensure_validated_4c1e(
    plan: &ExecutionPlan<'_>,
    specialization: &SpecializationKey,
) -> Result<(), cintxRsError> {
    // 1. Spinor rejection FIRST — before feature gate (D-05)
    if matches!(plan.representation, Representation::Spinor) {
        return Err(validated_4c1e_error("spinor representation not supported for 4c1e"));
    }

    // 2. Family sanity check
    if specialization.canonical_family() != "4c1e" {
        return Err(cintxRsError::ChunkPlanFailed { ... });
    }

    // 3. Cart/sph check (redundant after spinor, but explicit)
    if !matches!(plan.representation, Representation::Cart | Representation::Spheric) {
        return Err(validated_4c1e_error("representation must be cart/sph"));
    }

    // 4. Scalar component rank
    if !plan.descriptor.entry.component_rank.trim().is_empty()
        && plan.descriptor.entry.component_rank != "scalar" {
        return Err(validated_4c1e_error("component rank must be scalar"));
    }

    // 5. Max angular momentum
    if plan.shells.as_slice().iter().any(|s| s.ang_momentum > 4) {
        return Err(validated_4c1e_error("max(l)>4"));
    }

    Ok(())
}
```

**Step 2 — Implement fill_4c1e_g_tensor (D-03, from g4c1e.c):**

Add a private function that fills the G-tensor using the 4c1e polynomial recurrence. This replaces the Rys quadrature fill. The key algorithm:

```rust
fn fill_4c1e_g_tensor(
    g: &mut [f64],
    shape: &TwoEShape,  // Reuse TwoEShape from two_electron.rs — nroots forced to 1
    ri: [f64; 3], rj: [f64; 3], rk: [f64; 3], rl: [f64; 3],
    rij: [f64; 3], rkl: [f64; 3],
    aij: f64, akl: f64,
    fac: f64,
) {
    let aijkl = aij + akl;
    let nmax = shape.li + shape.lj;
    let mmax = shape.lk + shape.ll;

    for axis in 0..3usize {
        // Determine base center per g4c1e.c — follows ibase/kbase from shape
        let (r1, r2) = if nmax >= mmax {
            (if shape.ibase { ri[axis] } else { rj[axis] },
             if shape.kbase { rk[axis] } else { rl[axis] })
        } else {
            (if shape.kbase { rk[axis] } else { rl[axis] },
             if shape.ibase { ri[axis] } else { rj[axis] })
        };
        let rp = (aij * rij[axis] + akl * rkl[axis]) / aijkl;
        let r1r12 = r1 - rp;

        // 1D polynomial recurrence in scratch buffer
        let db = nmax + mmax + 1;
        let bigger = nmax.max(mmax);
        let mut buf = vec![0.0f64; db * (bigger + 1)];

        buf[0] = if axis == 2 { fac / (aijkl * aijkl.sqrt()) } else { 1.0 };
        if nmax + mmax > 0 {
            buf[1] = -r1r12 * buf[0];
        }
        for i in 1..(nmax + mmax) {
            buf[i + 1] = 0.5 * (i as f64) / aijkl * buf[i - 1] - r1r12 * buf[i];
        }

        // 2D shift fill: buf[j*db + i] for j>0
        let r1r2 = r1 - r2;
        for j in 1..=bigger {
            for i in 0..db.saturating_sub(j) {
                buf[j * db + i] = buf[(j - 1) * db + i + 1] + r1r2 * buf[(j - 1) * db + i];
            }
        }

        // Map to G-tensor: g[axis*g_size + i*dn + j*dm] where dn/dm come from shape
        // For nmax >= mmax: i loops 0..=nmax, j loops 0..=mmax, map = buf[j*db + i]
        // For nmax < mmax: swap i,j roles
        let off = axis * shape.g_size;
        if nmax >= mmax {
            for j in 0..=mmax {
                for i in 0..=nmax {
                    g[off + i * shape.dn + j * shape.dm] = buf[j * db + i];
                }
            }
        } else {
            for i in 0..=nmax {
                for j in 0..=mmax {
                    g[off + i * shape.dn + j * shape.dm] = buf[i * db + j];
                }
            }
        }
    }
}
```

CRITICAL: The fac factor `1/(aijkl * sqrt(aijkl))` is only applied on the z-axis (axis==2) initial value. The x and y axes start with `buf[0] = 1.0`. The final product of x*y*z contributions is formed during the contraction step, NOT in the fill. This matches g4c1e.c where `fac` is applied once to the z-component.

**Step 4 — Replace launch_center_4c1e stub:**

Replace the stub body with real kernel logic following the pattern from `launch_two_electron`:
1. Extract shell data (angular momenta, exponents, coefficients, centers) from `plan.shells`
2. Build 4c1e shape with `build_4c1e_shape(li, lj, lk, ll, aij, akl)`
3. Allocate G-tensor: `vec![0.0f64; 3 * shape.g_size]`
4. Primitive contraction loop over all (iprim, jprim, kprim, lprim) combinations:
   - Compute pair data: `aij = ai + aj`, `rij = (ai*ri + aj*rj) / aij`
   - Compute pair data: `akl = ak + al`, `rkl = (ak*rk + al*rl) / akl`
   - Compute contraction prefactor `fac` (overlap prefactors from ci*cj*ck*cl * gaussian normalization)
   - Call `fill_4c1e_g_tensor(g, shape, ri, rj, rk, rl, rij, rkl, aij, akl, fac)`
5. Apply HRR (same 4-branch from two_electron.rs):
   - If lj > 0: `hrr_lj2d_4d(g, shape, rirj, rkrl)`
   - If ll > 0: `hrr_kj2d_4d(g, shape, rirj, rkrl)` etc.
6. Contract G-tensor to Cartesian output: `contract_2e_cart(g, shape, li, lj, lk, ll)`
7. Apply c2s transform if `plan.representation == Spheric`
8. Copy result to staging buffer

The `_ = backend;` suppression is removed since the kernel now uses the plan data actively. Keep `let _ = backend;` since this is a host-side kernel (consistent with other host-side kernels in Phase 9/10).
