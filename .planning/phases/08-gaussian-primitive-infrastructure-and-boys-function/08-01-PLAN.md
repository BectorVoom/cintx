    Port libcint's `gamma_inc_like()` from `fmt.c` (lines 42-226) as a `#[cube]` function.

    Constants to embed at module level:
    - `MMAX: u32 = 39` (per D-02, matches last non-zero TURNOVER_POINT index)
    - `SQRTPIE4: f64 = 0.886226925452758013649083741670572591398774728061193564106903894926`
    - `DBL_EPSILON_HALF: f64 = f64::EPSILON * 0.5` (convergence tolerance)
    - `TURNOVER_POINT: [f64; 40]` array with the exact 40 values from fmt.c lines 8-17 (reproduced in 08-RESEARCH.md)

    Main function signature:
    ```rust
    #[cube]
    pub fn boys_gamma_inc(f: &mut Array<f64>, t: f64, m: u32)
    ```

    Algorithm (single function with internal branching per D-07):
    1. **Branch t == 0.0**: Fill `f[k] = 1.0 / (2*k + 1) as f64` for k=0..=m via while loop with u32 counter.
    2. **Branch t < TURNOVER_POINT[m]** (power series per fmt.c):
       - Compute `b = m as f64 + 0.5`
       - `e = 0.5 * f64::exp(-t)`
       - Iterate: `x = x * t / bi; s = s + x; bi = bi + 1.0` while `x > DBL_EPSILON_HALF * e` (use f64::abs on x for safety)
       - `f[m] = s / b`
       - Downward recurrence: `f[i-1] = (e + t * f[i]) / (i as f64 - 0.5)` for i from m down to 1
    3. **Branch t >= TURNOVER_POINT[m]** (erfc + upward recurrence per D-03):
       - `tt = f64::sqrt(t)`
       - `f[0] = SQRTPIE4 / tt * f64::erf(tt)` (per research, erf available on CPU backend)
       - `e = f64::exp(-t)`
       - `b = 0.5 / t`
       - Upward recurrence: `f[i] = b * ((2*i - 1) as f64 * f[i-1] - e)` for i=1..=m

    Use statement-form if/else (never if-expressions as values). All loop counters are `u32`.

    **CRITICAL CubeCL constraint:** If const array indexing `TURNOVER_POINT[m]` with runtime `m: u32` fails to compile, fall back to passing `turnover` as a pre-computed `f64` parameter from host side. The research identifies this as Open Question 2. Test compilation first; if it fails, add a host-side wrapper `boys_gamma_inc_host(f, t, m)` that looks up `TURNOVER_POINT[m]` and passes it to the `#[cube]` function.

    **erf fallback:** If `f64::erf()` is not available in CubeCL 0.9.x (compile error), implement a high-accuracy polynomial approximation as a separate `#[cube]` helper function `erf_approx(x: f64) -> f64` using the Abramowitz & Stegun 7.1.26 rational approximation (max error ~1.5e-7, sufficient for Boys 1e-12 when combined with the sqrt(pi/4)/sqrt(t) scaling). Document the approximation order used.
