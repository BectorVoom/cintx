    **Step 1 — Create `crates/cintx-cubecl/src/math/obara_saika.rs` per D-14:**

    Implement two core `#[cube]` functions matching the recurrence from g1e.c and g2e.c:

    **1. `vrr_step` — Vertical recurrence for one Cartesian dimension (1e overlap pattern per g1e.c CINTg1e_ovlp):**
    ```rust
    #[cube]
    pub fn vrr_step(
        g: &mut Array<f64>,  // G-array, indexed as g[n * stride]
        rijrx: f64,          // center displacement: rij[d] - rx[d]
        aij2: f64,           // 0.5 / (ai + aj)
        nmax: u32,           // maximum angular momentum on this center
        stride: u32,         // stride between consecutive n-indices in g
    )
    ```
    Recurrence (from g1e.c lines 130-150):
    - `g[1 * stride] = rijrx * g[0]`
    - For n = 1..nmax-1: `g[(n+1) * stride] = n as f64 * aij2 * g[(n-1) * stride] + rijrx * g[n * stride]`

    The g[0] value must be set before calling vrr_step (it's the base case, typically `fac * Boys_value` or 1.0 for testing).

    **2. `hrr_step` — Horizontal recurrence for one Cartesian dimension (per g1e.c, g2e.c HRR):**
    ```rust
    #[cube]
    pub fn hrr_step(
        g: &mut Array<f64>,  // G-array with both i and j indices
        rirj: f64,           // displacement Ri[d] - Rj[d]
        di: u32,             // stride along i-index
        dj: u32,             // stride along j-index
        li_max: u32,         // max angular momentum on center i
        lj: u32,             // target j angular momentum to build
    )
    ```
    Recurrence:
    - For each j from 1..=lj:
      - For each i from 0..=(li_max - j):
        - `g[j * dj + i * di] = g[(j-1) * dj + (i+1) * di] + rirj * g[(j-1) * dj + i * di]`

    **3. `vrr_2e_step` — Vertical recurrence for 2e integrals with Rys root-dependent coefficients (per g2e.c CINTg0_2e_2d):**
    ```rust
    #[cube]
    pub fn vrr_2e_step(
        g: &mut Array<f64>,
        c00: f64,    // Rys-weighted center displacement for bra
        b10: f64,    // recurrence coefficient bra-direction
        nmax: u32,   // max angular momentum bra
        stride: u32,
    )
    ```
    Same structure as vrr_step but with Rys root-specific coefficients c00 and b10 instead of rijrx and aij2. The recurrence is:
    - `g[1 * stride] = c00 * g[0]`
    - For n=1..nmax-1: `g[(n+1) * stride] = n as f64 * b10 * g[(n-1) * stride] + c00 * g[n * stride]`

    All functions use u32 loop counters. No recursion. All helpers are `#[cube]`.

    **Create `crates/cintx-cubecl/tests/math_integration_tests.rs` per D-17:**

    This test validates that all four math primitives chain correctly in a realistic scenario. It implements a simplified 1e overlap auxiliary integral evaluation pipeline.

    **Test: `math_integration_1e_overlap_ss`** (s-s shell pair, simplest case):

    Pipeline:
    1. Define a primitive pair: ai=1.0, aj=1.5, Ri=(0,0,0), Rj=(1.0,0,0), norm_i=1.0, norm_j=1.0
    2. Call `compute_pdata(ai, aj, ...)` to get PairData (zeta=2.5, center_p=(0.6,0,0), aij2=0.2, rirj=(-1,0,0), fac=exp(-0.6))
    3. Compute `t = 0.0` (for overlap, no nuclear attraction). For 1e overlap, the Boys argument is 0 since there is no operator center.
    4. Call `boys_gamma_inc(f, t=0, m=0)` -> f[0] = 1.0
    5. Set g[0] = pdata.fac * f[0] (base case for VRR)
    6. For s-s (lmax=0 on both centers): no VRR or HRR needed. Final integral = g[0] = fac.
    7. Verify: the s-s overlap integral for this pair equals `fac * (pi/zeta)^{3/2}` (Gaussian product theorem). Cross-check the computed g[0] against this analytical formula.

    **Test: `math_integration_1e_overlap_ps`** (p-s shell pair):

    Pipeline:
    1. Same primitive pair as above
    2. compute_pdata -> PairData
    3. boys_gamma_inc(f, t=0, m=1) -> f[0]=1.0, f[1]=1/3
    4. Set g[0] = pdata.fac * f[0]
    5. Call vrr_step(g, rijrx=center_p_x - ri_x, aij2, nmax=1, stride=1) -> builds g[1]
    6. Final p-s integral components = g[0] (s-component) and g[1] (p_x component)
    7. Verify g[1] is non-zero and consistent with analytical Gaussian integral formula for p-s overlap

    **Test: `math_integration_1e_overlap_ds`** (d-s shell pair):

    Same pipeline but with nmax=2 for vrr_step. Verify g[0], g[1], g[2] are all non-zero and self-consistent.

    **Test: `math_integration_rys_boys_crosscheck`:**

    For nroots=1 Rys quadrature: verify w[0] = F_0(x) by calling both rys_roots and boys_gamma_inc for the same x value and comparing. Test at x=1.0, 5.0, 15.0.
