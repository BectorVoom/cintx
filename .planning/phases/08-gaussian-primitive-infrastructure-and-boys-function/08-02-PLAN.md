```c
// CINTrys_roots(nroots, x, u, w) where:
//   nroots: number of quadrature points (1..12+)
//   x: argument (related to exponent ratio)
//   u[nroots]: output roots
//   w[nroots]: output weights
//
// Dispatch by nroots:
//   1: rys_root1(x, u, w)
//   2: rys_root2(x, u, w)
//   3: rys_root3(x, u, w)
//   4: rys_root4(x, u, w)
//   5: rys_root5(x, u, w)

```

```c
// _CINT_matmul_14_14 — applies COS_14_14 to coefficient vector
```

    **Core `#[cube]` functions:**

    1. `clenshaw_d1(coeffs: &Array<f64>, n: u32, x: f64) -> f64` — Clenshaw backward recurrence for Chebyshev polynomial evaluation. Port from `_CINT_clenshaw_d1` in polyfits.c:
       ```
       d1 = 0, d2 = 0
       for k = n-1 down to 1:
           d0 = 2*x*d1 - d2 + coeffs[k]
           d2 = d1; d1 = d0
       return x*d1 - d2 + 0.5*coeffs[0]
       ```

    2. `rys_root1(x: f64, u: &mut Array<f64>, w: &mut Array<f64>)` — Single-root Rys. For nroots=1 this is essentially the Boys function F_0/F_1 ratio. Port from rys_roots.c `rys_root1()`: uses polynomial fit coefficients for different x-ranges.

    3. `rys_root2(x: f64, u: &mut Array<f64>, w: &mut Array<f64>)` through `rys_root5(...)` — Port each dedicated function from rys_roots.c. Each has:
       - Domain segmentation (x ranges with different polynomial fits)
       - Polynomial fit coefficient arrays for roots and weights per segment
       - Clenshaw evaluation calls

    4. `rys_roots(nroots: u32, x: f64, u: &mut Array<f64>, w: &mut Array<f64>)` — Main dispatch function per D-10:
       ```rust
       #[cube]
       pub fn rys_roots(nroots: u32, x: f64, u: &mut Array<f64>, w: &mut Array<f64>) {
           if nroots == 1u32 {
               rys_root1(x, u, w);
           } else if nroots == 2u32 {
               rys_root2(x, u, w);
           } else if nroots == 3u32 {
               rys_root3(x, u, w);
           } else if nroots == 4u32 {
               rys_root4(x, u, w);
           } else if nroots == 5u32 {
               rys_root5(x, u, w);
           }
           // nroots > 5: Phase 10 will add Wheeler fallback (D-11)
       }
       ```

    **Polynomial coefficient extraction strategy:** Read `rys_roots.c` carefully for each `rys_rootN` function. Each one segments the x-domain and applies pre-computed polynomial fits. The coefficients are often inlined as static arrays in the C source. Transcribe these as `const` arrays in Rust at module level. There will be many coefficient arrays — this is expected and correct.

    **Size warning:** The polynomial coefficient tables from polyfits.c / rys_roots.c can be large (hundreds of f64 constants). This is intentional — they are compile-time constants with zero runtime overhead per D-09.

    **CubeCL constraints:**
    - All 5 `rys_rootN` functions and `clenshaw_d1` must be `#[cube]`
    - Use u32 for all loop counters
    - Statement-form if/else for domain segmentation
    - `f64::sqrt()`, `f64::exp()` for math operations
