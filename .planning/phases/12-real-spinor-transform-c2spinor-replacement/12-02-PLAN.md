    ```c
    FINT int1e_ovlp_spinor(double complex *out, FINT *dims, FINT *shls,
    ```

    CRITICAL: `double complex *out` is interleaved doubles (re, im) pairs. The output pointer is `*mut f64` with length `ni * nj * 2` where `ni = CINTcgto_spinor(shls[0])` and `nj = CINTcgto_spinor(shls[1])`.

    **Step 2: Add extern declarations** in the `ffi` module (or supplemental header):
    ```rust
    extern "C" {
        pub fn int1e_ovlp_spinor(
            out: *mut f64, dims: *mut i32, shls: *mut i32,
            atm: *mut i32, natm: i32, bas: *mut i32, nbas: i32,
            env: *mut f64, opt: *mut std::ffi::c_void, cache: *mut f64,
        ) -> i32;
        // ... same for int1e_kin_spinor, int1e_nuc_spinor
    }
    ```

    Note: `double complex *` maps to `*mut f64` since we want to read the interleaved representation directly. This is correct because `double complex` in C is layout-compatible with `[f64; 2]`.

    **Step 2: Build H2O STO-3G fixtures with spinor-compatible shells.**

    Reuse the existing `build_h2o_sto3g()` function. For spinor integrals, the shell `kappa` field matters:
    - s-shell (l=0): kappa = -1 (j=1/2, nd=2)
    - p-shell (l=1): kappa = -1 (j=3/2, nd=4) or kappa = +1 (j=1/2, nd=2)

    The existing bas array uses `kappa=0` for all shells (which means both j blocks, nd=4l+2). This is the general case and exercises both gt and lt coefficient blocks. Keep kappa=0 for initial testing.

    **Step 3: Implement the test function:**

    ```rust
    #[test]
    #[cfg(has_vendor_libcint)]
    fn oracle_gate_1e_spinor() {
        let (atm, bas, env) = build_h2o_sto3g();
        let natm = (atm.len() / ATM_SLOTS) as i32;
        let nbas = (bas.len() / BAS_SLOTS) as i32;

        // Test all shell pairs for 1e spinor operators
        let operators = [
            ("int1e_ovlp_spinor", vendor_int1e_ovlp_spinor as fn(...) -> i32),
            ("int1e_kin_spinor", vendor_int1e_kin_spinor as fn(...) -> i32),
            ("int1e_nuc_spinor", vendor_int1e_nuc_spinor as fn(...) -> i32),
        ];

        let mut total_mismatches = 0;

        for (name, vendor_fn) in &operators {
            for i in 0..nbas {
                for j in 0..nbas {
                    let shls = [i, j];
                    // Spinor component count: CINTcgto_spinor for each shell
                    let ni = vendor_CINTcgto_spinor(i, &bas);
                    let nj = vendor_CINTcgto_spinor(j, &bas);
                    // Output buffer: ni * nj * 2 (interleaved re/im)
                    let nelems = (ni * nj * 2) as usize;

                    let mut vendor_out = vec![0.0f64; nelems];
                    let mut cintx_out = vec![0.0f64; nelems];

                    // Vendor call
                    vendor_fn(&mut vendor_out, &shls, &atm, natm, &bas, nbas, &env, std::ptr::null_mut());

                    // cintx call via eval_raw with spinor RawApiId
                    // Use the appropriate spinor symbol RawApiId
                    let api_id = RawApiId::from_symbol(name).expect("spinor symbol must resolve");
                    let dims = [ni, nj];
                    eval_raw(api_id, &mut cintx_out, &dims, &shls, &atm, natm, &bas, nbas, &env, std::ptr::null_mut());

                    // Compare element-wise
                    for k in 0..nelems {
                        let diff = (cintx_out[k] - vendor_out[k]).abs();
                        if diff > ATOL_SPINOR {
                            eprintln!("{name} shls=[{i},{j}] elem {k}: cintx={} vendor={} diff={diff}", cintx_out[k], vendor_out[k]);
                            total_mismatches += 1;
                        }
                    }
                }
            }
        }

        assert_eq!(total_mismatches, 0, "1e spinor oracle parity: {total_mismatches} mismatches");
    }
    ```

    Adapt this pattern to match the actual API signatures used in the existing tests. Key things to verify:
