    **Part A: Add `cart_to_spinor_sf_4d` for 2e (4-center)**

    This implements the two-step libcint spinor transform for 4-center integrals:

    Step 1 (matching `c2s_sf_2e1`): Transform (i,j) bra/ket pair to spinor, keeping (k,l) as Cartesian.
    - Input: `cart[nci * ncj * nck * ncl]` in row-major order. The (i,j) indices are the inner dimensions, (k,l) outer.
    - For each (k,l) slice of size `nci * ncj`:
      - Apply `apply_bra_sf_block_all_kappa` on bra index i (over `ncj` columns), producing `tmp[di * ncj]` complex
      - Apply `apply_ket_transform` on ket index j, producing `out[di * dj]` complex
    - Intermediate result: `opij[di * nck * ncl * dj]` complex (interleaved)

    Step 2 (matching `c2s_sf_2e2`): Transform (k,l) pair to spinor.
    - The intermediate from step 1 has shape `[di * nck * ncl * dj]` as complex.
    - For this, we need to apply the bra spinor on k and ket spinor on l.
    - CRITICAL: Step 2 operates on complex data (output of step 1). The `a_bra1_cart2spinor_zf` function in libcint works on complex (zero-field) data where the spinor coefficient matrix is applied to both real and imaginary parts.
    - For each (i_sp, j_sp) spinor pair from step 1, apply bra on k-index and ket on l-index.
    - Output: `staging[di * dj * dk * dl * 2]` in column-major interleaved layout matching `zcopy_iklj`.

    Signature:
    ```rust
    pub fn cart_to_spinor_sf_4d(
        staging: &mut [f64],
        cart: &[f64],
        li: u8, kappa_i: i16,
        lj: u8, kappa_j: i16,
        lk: u8, kappa_k: i16,
        ll: u8, kappa_l: i16,
    ) -> Result<(), cintxRsError>
    ```

    The implementation must match libcint exactly. Key references:
    - `c2s_sf_2e1` at cart2sph.c line 5385: bra on i, ket on j, keeping k*l Cartesian columns
    - `c2s_sf_2e2` at cart2sph.c line 5469: bra on k (complex zero-field), ket on l
    - `zcopy_iklj` at cart2sph.c: copies with index permutation to final column-major layout

    The internal helpers `apply_bra_sf_block_all_kappa` and `apply_ket_transform` already exist from `cart_to_spinor_sf_2d`. Reuse them for both step 1 and step 2 (adapting stride parameters).

    For step 2, the bra transform operates on complex data. The `a_bra1_cart2spinor_zf` in libcint multiplies complex coefficient by complex input: `(cR + i*cI) * (vR + i*vI)`. This differs from step 1's `a_bra_cart2spinor_sf` which multiplies real coefficient by real input with conjugate sign on imaginary. Implement a separate `apply_bra_zf_block` function that handles complex-by-complex multiplication:
    ```
    out_R += cR * vR - cI * vI
    out_I += cR * vI + cI * vR
    ```
    Note the sign difference from `apply_bra_sf` which uses conjugate: `out_I += -cI * v`.

    The output layout follows `zcopy_iklj`: `staging[(((l_sp * nk + k_sp) * nj + j_sp) * ni + i_sp) * 2]` = re, +1 = im. This is column-major with spinor indices (i inner, j, k, l outer).

    Add unit tests:
    - Test with all s-shells (l=0, kappa=-1 for all four): simplest case, 2^4 = 16 spinor components x 2 = 32 f64 output
    - Test that output is non-zero (not just zeros from no-op)
    - Test that output size matches `di * dj * dk * dl * 2`
