    **Spinor integral C signatures (from libcint):**
    ```c
    FINT int2e_spinor(double complex *out, FINT *dims, FINT *shls,
                      FINT *atm, FINT natm, FINT *bas, FINT nbas, double *env,
                      CINTOpt *opt, double *cache);
    FINT int2c2e_spinor(double complex *out, FINT *dims, FINT *shls,
                        FINT *atm, FINT natm, FINT *bas, FINT nbas, double *env,
                        CINTOpt *opt, double *cache);
    FINT int3c1e_spinor(double complex *out, FINT *dims, FINT *shls,
                        FINT *atm, FINT natm, FINT *bas, FINT nbas, double *env,
                        CINTOpt *opt, double *cache);
    FINT int3c2e_spinor(double complex *out, FINT *dims, FINT *shls,
                        FINT *atm, FINT natm, FINT *bas, FINT nbas, double *env,
                        CINTOpt *opt, double *cache);
    ```

    Follow the exact same pattern as the 1e spinor wrappers from Plan 02. Key differences:
    - `int2e_spinor`: takes `shls: &[i32; 4]` (4-center), output size = `ni*nj*nk*nl*2`
    - `int2c2e_spinor`: takes `shls: &[i32; 2]` (2-center), output size = `ni*nj*2`
    - `int3c1e_spinor`: takes `shls: &[i32; 3]` (3-center), output size = `ni*nj*nk*2`
    - `int3c2e_spinor`: takes `shls: &[i32; 3]` (3-center), output size = `ni*nj*nk*2`

    Where `ni = CINTcgto_spinor(shls[0])`, etc. The `*2` accounts for interleaved re/im.

    If any of these functions are not declared in the libcint headers, add them to the supplemental bindgen header in build.rs (same approach as Phase 10 for `int2c2e_sph` etc.).

    Create safe wrappers:
    ```rust
    #[cfg(has_vendor_libcint)]
    pub fn vendor_int2e_spinor(
        out: &mut [f64], shls: &[i32; 4], atm: &[i32], natm: i32,
        bas: &[i32], nbas: i32, env: &[f64], opt: *mut std::ffi::c_void,
    ) -> i32 { ... }
    ```
    Repeat for `vendor_int2c2e_spinor`, `vendor_int3c1e_spinor`, `vendor_int3c2e_spinor`.
