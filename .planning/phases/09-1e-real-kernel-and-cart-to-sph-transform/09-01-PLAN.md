    3. **Transform function** `pub fn cart_to_sph_1e(cart_buf: &[f64], sph_buf: &mut [f64], li: u8, lj: u8)`:
       - Input `cart_buf`: flat array of shape `[ncart_j * ncart_i]` where `ncart = (l+1)(l+2)/2`
       - Output `sph_buf`: flat array of shape `[nsph_j * nsph_i]` where `nsph = 2l+1`
       - Apply bra transform (T[li]) and ket transform (T[lj]) per research Pattern 4:
         Step 1: Transform bra (i-axis): for each j_cart column, multiply `T[li]` times the i_cart column to get i_sph values
         Step 2: Transform ket (j-axis): for each i_sph row, multiply `T[lj]` times the j_cart row to get j_sph values
       - For l=0, T is identity (1x1), so no-op on that axis.
       - For l=1, the p-shell transform extracts from libcint's g_trans_cart2sph[1..10].

    4. **Replace `cart_to_spheric_staging`** to be a no-op:
       ```rust
       pub fn cart_to_spheric_staging(staging: &mut [f64]) -> Result<(), cintxRsError> {

           let _ = staging;
           Ok(())
       }
       ```
       This ensures the executor's `apply_representation_transform(Spheric, staging)` call does not corrupt kernel output. Delete the old running-blend stub and its unit test.
