    **Part B: Replace `launch_one_electron` in `crates/cintx-cubecl/src/kernels/one_electron.rs`**

    Replace the entire stub with a real host-side pipeline. The function signature stays the same:
    ```rust
    pub fn launch_one_electron(
        backend: &ResolvedBackend,
        plan: &ExecutionPlan<'_>,
        specialization: &SpecializationKey,
        staging: &mut [f64],
    ) -> Result<ExecutionStats, cintxRsError>
    ```

    Implementation structure (per D-01, D-02, D-03):

    1. **Extract shell info** from `plan` and `specialization`:
       - `li = specialization.shell_angular_momentum()[0]` (bra angular momentum)
       - `lj = specialization.shell_angular_momentum()[1]` (ket angular momentum)
       - Shell exponents, coefficients, centers from `plan.shells.as_slice()`
       - Each shell has: `ang_momentum`, `nprim` (number of primitives), `nctr` (contractions), 
         exponents via `exponents()`, coefficients via `coefficients()`, center from the associated atom

    2. **Determine operator** from `plan.descriptor.operator_name()` (per Claude's discretion):
       - `"overlap"` -> overlap operator
       - `"kinetic"` -> kinetic operator  
       - `"nuclear-attraction"` -> nuclear attraction operator
       - Anything else -> return `UnsupportedApi` error

    3. **Compute integrals** per primitive pair (ai, aj):
       For each contraction pair (ci, cj) and primitive pair (ai, aj):

       a. Call `compute_pdata_host(ai, aj, ri_x, ri_y, ri_z, rj_x, rj_y, rj_z, norm_i, norm_j)` to get PairData.

       b. **G-tensor fill** (per D-03, g1e.c lines 127-182):
          - Allocate g-array: `vec![0.0; 3 * (li + lj + 1)]` — three axes [gx | gy | gz]
          - Set base case: `gx[0] = 1.0`, `gy[0] = 1.0`, `gz[0] = fac * SQRTPI * PI / (aij * sqrt(aij))`
            where `fac = pd.fac`, `aij = pd.zeta_ab`
          - The constant is `SQRTPI * PI / (aij * sqrt(aij))` NOT `SQRTPI * M_PI / (aij * sqrt(aij))` — use `1.7724538509055159 * std::f64::consts::PI / (aij * aij.sqrt())`
          - VRR fill for each axis using `vrr_step_host`:
            * Determine `ibase`: if `li >= lj`, VRR runs on i-center first (rijrx = center_p - ri), else on j-center (rijrx = center_p - rj)
            * `nmax = li + lj` for VRR
            * stride = 1 for the flat per-axis array
            * Call `vrr_step_host(&mut gx, rijrx_x, pd.aij2, nmax, 1)` for x-axis
            * Same for gy (rijrx_y) and gz (rijrx_z, but gz[0] carries the norm factor)
          - HRR transfer if `lj > 0`: call `hrr_step_host` to shift angular momentum from VRR center to j-center
            * di = 1, dj = li + lj + 1 (or nmax + 1, to avoid overwriting)
            * Actually, for 1e the g-array strides need careful layout: see g1e.c envs->g_stride_i and envs->g_stride_j
            * For the initial host implementation: use g_size = (nmax + 1) per axis, di = 1, dj = (nmax + 1)
            * The g-array per axis needs size `(li + 1) * (lj + 1)` after HRR, so allocate `(nmax + 1) * (lj + 1)` per axis
            * Actually for simplicity: allocate `g_per_axis = (li + lj + 1) * max(lj + 1, 1)` elements

       c. **Operator-specific contraction** (per D-02):

          **Overlap** (`CINTgout1e` pattern, cint1e.c lines 306-325):
          ```
          For each (ix, iy, iz) cartesian index triple where ix+iy+iz = li and jx+jy+jz = lj:
            gout[n] = gx[ix + jx*dj] * gy[iy + jy*dj] * gz[iz + jz*dj]
          ```
          The cartesian index enumeration follows `CINTcart_comp(int *nx, int *ny, int *nz, int lmax)`:
          For l, iterate `(nx, ny, nz)` as: for lx=l..0, for ly=l-lx..0, nz=l-lx-ly.

          **Kinetic** (per D-02, intor1.c lines 18-46):
          - G-array must be computed with `nmax = li + lj + 2` (extra angular momentum for derivative)
          - Apply `nabla_j` derivative twice (D_J composed twice, not just once):
            * `g1[n] = nabla_j(g0, n)` where `nabla_j(g, n) = -2*aj * g[n + dj]` (j=0) or `j * g[n-dj] - 2*aj * g[n+dj]` (j>0)
            * `g3[n] = nabla_j(g1, n)` (second application, but this is on g1 which already has derivative)
            * Actually per research: `g2 = D_J(g0)` at lj+1 stride, `g3 = D_J(g2)` at lj stride
          - Contract: `gout[n] = -(g3x[ix]*g0y[iy]*g0z[iz] + g0x[ix]*g3y[iy]*g0z[iz] + g0x[ix]*g0y[iy]*g3z[iz])`
          - Apply `* 0.5` common factor to the final integral (not inside VRR)

          **Nuclear attraction** (per D-07, D-08, g1e.c lines 208-320):
          - `nrys_roots = (li + lj) / 2 + 1`
          - For each atom center C in `plan.basis.atoms()` with charge Z_C and position [cx, cy, cz]:
            * `crij[d] = C[d] - P[d]` where P = Gaussian product center (pd.center_p_x/y/z)
            * `x = pd.zeta_ab * (crij_x^2 + crij_y^2 + crij_z^2)` (Boys argument)
            * Get Rys roots/weights: call `rys_root1_host(x)` if nrys_roots==1, or `rys_root2_host(x)` if nrys_roots==2
            * For each Rys root n (0..nrys_roots):
              - `ru = u[n] / (1.0 + u[n])` (for point nucleus, tau=1)
              - `rt = pd.aij2 - pd.aij2 * ru` (modified recurrence coefficient)
              - `r0_x = (pd.center_p_x - rx_x) + ru * crij_x` (modified displacement per axis)
              - `fac1 = 2.0 * PI * (-Z_C) * pd.fac / pd.zeta_ab` (point nucleus prefactor)
              - `gz_root = fac1 * w[n]` (weight for this root)
              - Fill VRR per root: gx_root[0]=1, gy_root[0]=1, gz_root[0]=fac1*w[n]
              - Apply VRR with modified coefficients (c00=r0_x, b10=rt for the root-dependent recurrence)
                Actually the nuclear VRR uses `vrr_2e_step_host` pattern from obara_saika.rs, not `vrr_step_host` because the recurrence coefficient is root-dependent
              - HRR if needed
              - Contract and accumulate: `gout[n] += gx[ix]*gy[iy]*gz[iz]` over roots
            * Multiply by Z_C (nuclear charge) and accumulate over atoms

       d. **Contract over primitives**: Multiply each primitive pair contribution by `ci * cj` (contraction coefficients) and sum.

       e. **Cart-to-sph transform** if `plan.representation == Representation::Spheric` (per D-04):
          - The kernel writes cartesian integrals to a local buffer
          - Call `c2s::cart_to_sph_1e(&cart_buf, &mut staging[..nsph_total], li, lj)` to transform into staging
          - If Cart representation, copy cartesian directly to staging
          - `staging` is pre-sized for the target representation by the planner

       f. **Return ExecutionStats** with:
          - `not0 = staging.iter().filter(|&&v| v.abs() > 1e-18).count() as i32`
          - Other fields from workspace query as before

    **Important implementation notes:**
    - Use `crate::math::pdata::compute_pdata_host`
    - Use `crate::math::obara_saika::{vrr_step_host, hrr_step_host}`
    - Use `crate::math::rys::{rys_root1_host, rys_root2_host}`
    - Use `crate::transform::c2s::{cart_to_sph_1e, ncart, nsph}`
    - Shell exponents: `shell.exponents()` returns `&[f64]`
    - Shell coefficients: `shell.coefficients()` returns `&[f64]`
    - Atom center: `plan.basis.atoms()[shell.atom_index].coords()` returns `[f64; 3]`
    - Atom charge: `plan.basis.atoms()[atom_idx].atomic_number()` as f64 for Z
    - The `_ = backend;` suppression stays since the host-side pipeline doesn't use the backend directly (CPU backend executes Rust natively)
    - Keep `let _ = backend;` and `let _ = specialization;` if specialization fields are accessed via plan instead
    
    **Access to shell_angular_momentum:** The `SpecializationKey` has `shell_angular_momentum` as a private field. Access angular momentum from `plan.shells.as_slice()` instead:
