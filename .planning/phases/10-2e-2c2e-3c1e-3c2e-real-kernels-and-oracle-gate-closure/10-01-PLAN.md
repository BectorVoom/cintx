**Part A: c2s.rs — Add four multi-index cart-to-sph transform functions.**

All use the existing `C2S_L0`..`C2S_L4` coefficient tables and `c2s_coeff(l, sph_comp, cart_comp)` helper. The transform applies the c2s matrix along each index axis sequentially, using intermediate buffers.

1. **`cart_to_sph_2c2e(cart: &[f64], li: u32, lk: u32) -> Vec<f64>`** — 2-index (i, k) transform. Same structure as `cart_to_sph_1e` but with indices named i, k instead of i, j. Buffer shape: `[ncart(li), ncart(lk)]` -> `[nsph(li), nsph(lk)]`. Transform k-axis first into intermediate `[ncart(li), nsph(lk)]`, then i-axis.

2. **`cart_to_sph_3c1e(cart: &[f64], li: u32, lj: u32, lk: u32) -> Vec<f64>`** — 3-index (i, j, k) transform. Buffer shape: `[ncart(li), ncart(lj), ncart(lk)]` -> `[nsph(li), nsph(lj), nsph(lk)]`. Transform k-axis first, then j-axis, then i-axis.

3. **`cart_to_sph_3c2e(cart: &[f64], li: u32, lj: u32, lk: u32) -> Vec<f64>`** — 3-index (i, j, k) transform. Same as 3c1e transform (identical index structure).

4. **`cart_to_sph_2e(cart: &[f64], li: u32, lj: u32, lk: u32, ll: u32) -> Vec<f64>`** — 4-index (i, j, k, l) transform. Buffer shape: `[ncart(li), ncart(lj), ncart(lk), ncart(ll)]` -> `[nsph(li), nsph(lj), nsph(lk), nsph(ll)]`. Transform l-axis first, then k, then j, then i. Use row-major ordering where i is the slowest-varying index (outermost) and l is the fastest-varying (innermost).

Note: libcint output uses column-major ordering (i fastest). The c2s functions must match whichever ordering the kernel writes. The 1e kernel's `cart_to_sph_1e` uses libcint ordering (i fastest, j slowest = column-major). Follow the same convention for all families.
