Replace the stub in center_3c1e.rs with a real 3c1e overlap kernel (per D-01, D-03, D-12, D-13). The 3c1e family uses a three-center Gaussian product overlap VRR — NOT Rys quadrature (per Pitfall 5, Research Pattern 2).

**Algorithm from g3c1e.c CINTg3c1e_ovlp:**

The 3c1e integral involves shells i, j, k at centers A, B, C. The overlap integral computes the three-center Gaussian product:
1. Form the ij pair product: P = (ai*A + aj*B) / (ai+aj), aij = ai+aj
2. Form the ijk triple product: G = (aij*P + ak*C) / (aij+ak), aijk = aij+ak
3. The G-tensor has layout from CINTinit_int3c1e_EnvVars:
   ```
   g_stride_i = 1
   g_stride_j = li_ceil + 1
   g_stride_k = (li_ceil + 1) * (lj_ceil + lk_ceil + 1)
   g_size = (li_ceil + 1) * (lj_ceil + lk_ceil + 1) * (lk_ceil + 1)
   ```
   Note: the j dimension is `lj_ceil + lk_ceil + 1` (combined) because VRR builds j+k levels together, then HRR separates them.

4. **G-tensor fill (CINTg3c1e_ovlp):**
   - `gz[0, 0, 0] = fac * exp(-rho_ij * |A-B|^2 - rho_jk * ...) * (pi/aijk)^(3/2)` — the base case carries the three-center Gaussian overlap prefactor.
     More precisely: `gz_base = fac * sqrt(PI^3 / aijk^3) * exp(overlap_exponent)` where the overlap exponent combines all three pair distances.
   - `gx[0, 0, 0] = 1.0`, `gy[0, 0, 0] = 1.0`
   - VRR upward in i-direction (g1e-type): use `vrr_step_host` with `d_axis = P[axis] - A[axis]` and `b10 = 0.5 / aij`.
   - VRR upward in combined j+k direction: use `vrr_step_host` with center displacement `G[axis] - P[axis]` and `b10 = 0.5 / aijk`. Build up to `lj + lk` levels.
   - Then apply HRR to split the j+k dimension into separate j and k indices: `g[i, j, k+1] = g[i, j+1, k] + (B - C)[axis] * g[i, j, k]`.

   Actually, looking at g3c1e.c more carefully:
   - The g1e-style approach: first VRR builds i from 0 to li using center P-A displacement.
   - Then VRR builds the "j" dimension from 0 to lj+lk using a different center displacement (depends on the three-center product geometry).
   - Then HRR separates the combined j index into actual j and k components using `(C - B)` displacement.

5. **Primitive triple loop** over ip=0..nprim_i, jp=0..nprim_j, kp=0..nprim_k:
   Each iteration fills a G-tensor, contracts Cartesian components, accumulates with `ci * cj * ck` contraction coefficients.

6. **Contract** Cartesian components: for each `(ci_idx, cj_idx, ck_idx)` in `cart_comps(li) x cart_comps(lj) x cart_comps(lk)`:
   `cart_buf[ci_idx, cj_idx, ck_idx] += ci*cj*ck * gx[ix]*gy[iy]*gz[iz]`

7. Apply `common_fac_sp(li) * common_fac_sp(lj) * common_fac_sp(lk)` scaling.

8. If Spheric: call `cart_to_sph_3c1e(cart_buf, li, lj, lk)`.

9. Write to staging, return ExecutionStats.

**G-tensor base case normalization for 3c1e overlap:**
From g3c1e.c: `common_fac = sqrt(PI) * PI * fac_sp(li) * fac_sp(lj) * fac_sp(lk)`
The per-primitive contribution: `gz[0] = w * (PI/aijk)^(1.5) * exp(-ai*aj/(ai+aj) * |A-B|^2 - aij*ak/(aij+ak) * |P-C|^2)`

**Output ordering:** i fastest, k slowest (column-major, matching libcint).

**Key difference from 1e:** Three centers, three primitive loops, combined j+k VRR dimension followed by HRR separation. No Rys quadrature.
