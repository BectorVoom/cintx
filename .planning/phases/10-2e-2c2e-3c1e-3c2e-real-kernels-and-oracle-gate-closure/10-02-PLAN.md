**Structure of `launch_center_2c2e`:**
1. Extract 2 shells (i, k) and their atom coordinates from `plan`.
2. Determine `li`, `lk` angular momenta, `nprim_i`, `nprim_k`, contraction coefficients.
3. Compute `nrys_roots = (li + lk) / 2 + 1` (from g2c2e.c — reuses 2e formula with lj=ll=0).
4. Compute output sizing: `nfi = ncart(li)`, `nfk = ncart(lk)` for cart; `nsph_i`, `nsph_k` for sph.
5. Allocate cart_buf `[nfi * nfk]` initialized to zero.
6. **Primitive double loop** over `ip` in 0..nprim_i, `kp` in 0..nprim_k:
   a. Get exponents `ai`, `ak` and contraction coefficients `ci`, `ck`.
   b. Compute PairData for the i-k pair: `compute_pdata_host(ai, ak, ri, rk)`.
   c. Compute Rys argument: `x_rys = pdata.zeta * |center_p - rk|^2` ... Actually for 2c2e: the Rys argument is `aij * akl / (aij + akl) * |P - Q|^2` where P is the i center, Q is the k center, aij = ai, akl = ak (since j_l = l_l = 0, the pair is just the single primitive). Specifically: `rho = ai * ak / (ai + ak)`, `x_rys = rho * |ri - rk|^2`.
   d. Call `rys_roots_host(nrys_roots, x_rys)` to get roots `u[]` and weights `w[]`.
   e. **G-tensor fill per Rys root:** For each root `r` in 0..nrys_roots:
      - Compute the 2e-type recurrence coefficients from the Rys root `u[r]`:
        - `t2 = u[r]` (Rys root is t^2)
        - For each axis (x, y, z):
          - `b00 = 0.5 * t2 / (ai + ak)` (cross-center recurrence)
          - `b10 = (1.0 - t2 / (ai + ak) * ak) / (2.0 * ai)` (bra-bra recurrence, adjusted for 2e)
          - `b01 = (1.0 - t2 / (ai + ak) * ai) / (2.0 * ak)` (ket-ket recurrence, adjusted for 2e)
          
        Actually, follow the g2e.c CINTg0_2e_2d recurrence more carefully. The key variables from g2e.c:
        - `aij = ai`, `akl = ak` (2c2e has only one primitive per pair)
        - `a0 = aij * akl / (aij + akl)`
        - `a1 = aij / (aij + akl)` (factor for Q-shift)  
        - For each Rys root t^2 = u[r]:
          - `fac1 = 0.5 / (aij * (1 - t2/(aij+akl)))`... 
          
        The exact recurrence from g2e.c `CINTg0_2e_2d`:
        ```
        b10 = 0.5 / aij * (1 - akl*t2 / (aij+akl)) for each root
        b01 = 0.5 / akl * (1 - aij*t2 / (aij+akl)) for each root  
        b00 = 0.5 * t2 / (aij+akl)
        c00[axis] = P[axis] - ri[axis] + b00 * (Q[axis] - P[axis])  // shift P toward Q
        c0p[axis] = Q[axis] - rk[axis] + b10_ratio * (P[axis] - Q[axis]) // shift Q toward P
        ```
        where P = center of i gaussian = ri (since only one prim), Q = center of k gaussian = rk.
        Wait, for 2c2e with j_l=l_l=0: P is the Gaussian product center of shell i (which is just ri since there's no j), Q is rk.
        
        Actually: `aij = ai` (since j has zero exponent in 2c2e), `akl = ak`. The pair product center P = ri (single atom), Q = rk. Then:
        - `b10[r] = 0.5 * (1.0 - akl * u[r] / (aij + akl)) / aij`
        - `b01[r] = 0.5 * (1.0 - aij * u[r] / (aij + akl)) / akl`
        - `b00[r] = 0.5 * u[r] / (aij + akl)`
        - `c00[r][axis] = ri[axis] - ri[axis] + b00[r] * (rk[axis] - ri[axis])` = `b00[r] * (rk[axis] - ri[axis])`
        - `c0p[r][axis] = rk[axis] - rk[axis] - b01[r]/b00[r] * ...` No, use the standard formula:
          `c0p[r][axis] = Q[axis] - rk[axis] + akl/(aij+akl) * u[r] * (P[axis] - Q[axis])` ... 
          
        Follow the libcint source verbatim for the 2D recurrence. The key G-tensor layout for 2c2e:
        ```
        g_size = nrys_roots * (li+1) * (lk+1)
        g[root + i_level * nrys_roots + k_level * nrys_roots * (li+1)]
        ```
        Flat array is `[gx | gy | gz]`, total 3 * g_size.
        
      - Fill `gz[root, 0, 0] = w[r] * common_fac_2c2e` where `common_fac_2c2e = (PI^3 * 2 / sqrt(PI)) / ((aij + akl) * sqrt(aij + akl)) * pdata.fac`.
        Wait, the normalization for 2c2e from g2e.c line 54: `fac1 = (M_PI*M_PI*M_PI)*2/SQRTPI * CINTcommon_fac_sp(i_l) * CINTcommon_fac_sp(k_l)`. This is applied once. The Rys weight `w[r]` and the exponential factor from the Gaussian overlap are incorporated into gz[0].
        
        More precisely, from g2e.c `CINTg0_2e`:
        `s[r] = PI^(3/2) / (aij * akl * sqrt(aij+akl)) * exp(-rho * |ri-rk|^2) * w[r]`
        And gz[root] = s[r]. Then gx[root] = 1.0, gy[root] = 1.0.
        
      - Apply VRR upward in i-direction using b10, c00 coefficients.
      - Apply VRR upward in k-direction using b01, c0p coefficients.
      - (No HRR needed since j_l = l_l = 0.)
   f. **Contract Cartesian components:** Loop over cart_comp(li) x cart_comp(lk), sum `gx[ix] * gy[iy] * gz[iz]` products for each root, accumulate with `ci * ck` contraction coefficients.
7. Apply `common_fac_sp(li) * common_fac_sp(lk)` scaling (per Pitfall 2).
8. If `Spheric`: call `cart_to_sph_2c2e(cart_buf, li, lk)` and write result to staging.
9. If `Cart`: write cart_buf directly to staging.
10. Return `ExecutionStats` with appropriate values.

**Critical details:**

- The G-tensor base: `gz[root] = w[root] * fac` where `fac = PI^(3/2) / (ai * ak * sqrt(ai + ak)) * exp(-rho * |ri-rk|^2)`. But the `exp(-rho*R^2)` is already in `pdata.fac` from `compute_pdata_host`. So `gz[root] = w[root] * pdata.fac * PI^(3/2) / (ai * ak * sqrt(ai + ak))` ... verify against g2e.c exactly.
- Output ordering must match libcint column-major: i fastest, k slowest.
- Buffer lifecycle stays inside the function (per D-13).
