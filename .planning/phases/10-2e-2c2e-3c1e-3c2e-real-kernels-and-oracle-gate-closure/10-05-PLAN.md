  <action>
Replace the stub in two_electron.rs with a real 2e ERI kernel (per D-01, D-02, D-03, D-12, D-13).

**This is the most complex kernel. Follow g2e.c closely.**

**Structure of `launch_two_electron`:**

1. Extract 4 shells (i, j, k, l) and their atom coordinates (A, B, C, D) from `plan`.
2. Determine `li, lj, lk, ll` angular momenta and primitive counts/coefficients.
3. Compute `nrys = (li + lj + lk + ll) / 2 + 1`.
4. **Adaptive stride selection (Pitfall 1):**
   ```
   ibase = li >= lj  (simplified: always VRR on the larger of bra pair)
   kbase = lk >= ll  (same for ket pair)
   if ibase:
     di = li + 1, dj = lj + 1  // VRR builds i, HRR transfers to j
   else:
     di = lj + 1, dj = li + 1  // VRR builds j, HRR transfers to i
   if kbase:
     dk = lk + 1, dl = ll + 1
   else:
     dk = ll + 1, dl = lk + 1
   ```
   For simplicity, can always use `ibase = true, kbase = true` if that makes indexing clearer, but then need `nmax = max(li, lj) + lj + max(lk, ll) + ll` which may be unnecessarily large. Follow libcint's choice for minimal G-tensor size.

5. **G-tensor layout:**
   ```
   g_stride_i = nrys
   g_stride_k = nrys * di
   g_stride_l = nrys * di * dk
   g_stride_j = nrys * di * dk * dl
   g_size = nrys * di * dk * dl * dj
   ```
   Flat array: `[gx | gy | gz]`, total `3 * g_size`.

6. **Primitive quadruple loop** over ip, jp, kp, lp:
   a. Form ij pair: `aij = ai + aj`, `P = (ai*A + aj*B) / aij`
   b. Form kl pair: `akl = ak + al`, `Q = (ak*C + al*D) / akl`
   c. Compute `rho = aij * akl / (aij + akl)`, `x_rys = rho * |P - Q|^2`
   d. Get Rys roots/weights: `rys_roots_host(nrys, x_rys)`
   e. For each Rys root t^2 = u[r]:
      - Compute 2e recurrence coefficients (from CINTg0_2e_2d):
        - `b10 = 0.5 * (1 - akl * u[r] / (aij + akl)) / aij` (bra recurrence)
        - `b01 = 0.5 * (1 - aij * u[r] / (aij + akl)) / akl` (ket recurrence)
        - `b00 = 0.5 * u[r] / (aij + akl)` (cross recurrence)
        - `c00[axis] = P[axis] - A[axis] + (akl*u[r]/(aij+akl)) * (Q[axis]-P[axis]) / aij * akl`
          Actually: `c00[axis] = P[axis] - ri[axis] + b00 * (Q[axis] - P[axis])` where ri = A if ibase, else B
        - `c0p[axis] = Q[axis] - rk[axis] + b00 * (P[axis] - Q[axis])` where rk = C if kbase, else D
      - Fill gz base: `gz[root] = w[r] * PI^(3/2) / (aij * akl * sqrt(aij + akl)) * exp_ij * exp_kl`
        where `exp_ij = exp(-ai*aj/aij * |A-B|^2)`, `exp_kl = exp(-ak*al/akl * |C-D|^2)`.
        Use `compute_pdata_host` for both pairs to get `fac_ij` and `fac_kl`.
      - VRR upward in i/j direction (whichever is ibase) using b10, c00
      - VRR upward in k/l direction (whichever is kbase) using b01, c0p
      - Cross-term b00 handles coupled i-k recurrence
      - HRR to transfer from ibase direction to the other bra index
      - HRR to transfer from kbase direction to the other ket index
   f. **Contract Cartesian components:** Loop over `cart_comps(li) x cart_comps(lj) x cart_comps(lk) x cart_comps(ll)`:
      `cart_buf[ci,cj,ck,cl] += ci*cj*ck*cl * sum_roots(gx[ix]*gy[iy]*gz[iz])`
7. Apply `common_fac_sp(li) * common_fac_sp(lj) * common_fac_sp(lk) * common_fac_sp(ll)` (Pitfall 2 — all four factors).
8. If Spheric: call `cart_to_sph_2e(cart_buf, li, lj, lk, ll)`.
9. Write to staging, return ExecutionStats.

**Output ordering:** `out[i + j*nfi + k*nfi*nfj + l*nfi*nfj*nfk]` — i fastest, l slowest (column-major matching libcint, per Pitfall 7).

**VRR + HRR details:**
The 2e G-tensor fill uses vrr_2e_step_host for the root-dependent recurrence. For each Rys root, the VRR builds g[i, 0, k, 0] (if ibase and kbase) up from the base case, using:
- i-direction: `g[i+1] = c00 * g[i] + i * b10 * g[i-1]` (per axis)
- k-direction: `g[k+1] = c0p * g[k] + k * b01 * g[k-1]` (per axis)
- Cross: `g[i+1, k] += k * b00 * g[i, k-1]` (couples i and k raising)

Then HRR (no root dependence):
- j-direction: `g[i, j+1, k, l] = g[i+1, j, k, l] + (A-B)[axis] * g[i, j, k, l]`
- l-direction: `g[i, j, k, l+1] = g[i, j, k+1, l] + (C-D)[axis] * g[i, j, k, l]`

The HRR can run outside the Rys root loop since it only needs Cartesian differences.
