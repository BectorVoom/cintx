The ip1 derivative contraction uses G2E_D_I (= CINTnabla1i_2e) on the G tensor to produce
a derivative G tensor g1, then contracts 3 components:
  s[0] = sum_roots g1[ix+i]*g0[iy+i]*g0[iz+i]   (d/dx)
  s[1] = sum_roots g0[ix+i]*g1[iy+i]*g0[iz+i]   (d/dy)
  s[2] = sum_roots g0[ix+i]*g0[iy+i]*g1[iz+i]   (d/dz)
Output: gout[n*3 + comp] for each basis function tuple n and component comp.

The c2s_sph_2e1 transform is applied once per component in the CINT2e_drv driver:

**Part A: Add nabla derivative operations to f12.rs**

Port the three nabla functions from libcint g2e.c as host-side Rust functions:

1. `nabla1i_2e(f: &mut [f64], g: &[f64], li: usize, lj: usize, lk: usize, ll: usize, ai: f64, shape: &F12Shape)`:
   - f and g are both 3*g_size arrays (gx,gy,gz layout)
   - For each axis (0..3), offset by `axis * g_size`
   - Outer loops: j in 0..=lj, l in 0..=ll, k in 0..=lk
   - ptr = dj*j + dl*l + dk*k
   - i=0 case: for n in ptr..ptr+nroots: f[n] = ai2 * g[n+di] where ai2 = -2*ai
   - i>=1 case: for n in ptr..ptr+nroots: f[n] = i * g[n-di] + ai2 * g[n+di]
   - Stride di = shape.di (which equals nroots)
   - p1 = g offset by -di, p2 = g offset by +di

2. `nabla1j_2e(f, g, li, lj, lk, ll, aj, shape)`: Same pattern but stride=dj, exponent=aj.
   - Outer loops: l in 0..=ll, k in 0..=lk, inner i in 0..=li
   - j=0 case: f[n] = aj2 * g[n+dj]
   - j>=1 case: f[n] = j * g[n-dj] + aj2 * g[n+dj]

3. `nabla1k_2e(f, g, li, lj, lk, ll, ak, shape)`: Same pattern but stride=dk, exponent=ak.
   - Outer loops: j in 0..=lj, l in 0..=ll, inner i in 0..=li
   - k=0 case: f[n] = ak2 * g[n+dk]
   - k>=1 case: f[n] = k * g[n-dk] + ak2 * g[n+dk]

**Part B: Add per-variant gout contraction functions**

Each gout function takes the G tensor and produces ncomp * nf Cartesian values. Implement these
four functions matching libcint autocode exactly:

1. `gout_ip1(g: &[f64], shape: &F12Shape, li: usize, lj: usize, lk: usize, ll: usize, ai: f64) -> Vec<f64>`:
   - Allocate g1 = vec![0.0; 3*g_size]
   - Call nabla1i_2e(&mut g1, g, li+0, lj, lk, ll, ai, shape)
   - Note: the G tensor `g` was built with li_ceil=li+1 (F12_IP1.i_inc=1), so g has indices up to li+1. nabla1i_2e at li+0 reads g at index i+1 which is li+1 = the maximum available.
   - For each basis function tuple (idx), contract 3 components:
     s[0] = sum_roots g1[gx+ix+r]*g[gy+iy+r]*g[gz+iz+r]
     s[1] = sum_roots g[gx+ix+r]*g1[gy+iy+r]*g[gz+iz+r]
     s[2] = sum_roots g[gx+ix+r]*g[gy+iy+r]*g1[gz+iz+r]
   - Write to gout[n*3+comp] for comp in 0..3
   - The index computation for (ix,iy,iz) uses the same cart_comps iteration as contract_f12_cart but indexing at the base (non-ceil) angular momenta li,lj,lk,ll.

2. `gout_ipip1(g, shape, li, lj, lk, ll, ai) -> Vec<f64>`:
   - g1 = nabla1i_2e(g, li+1, lj, lk, ll, ai) — derivative at elevated i_l+1
   - g2 = nabla1i_2e(g, li+0, lj, lk, ll, ai) — derivative at base i_l
   - g3 = nabla1i_2e(g1, li+0, lj, lk, ll, ai) — second derivative
   - Contract 9 components per basis function:

**Part C: Modify f12_kernel_core to handle ncomp > 1**

The current `f12_kernel_core` computes a single Cartesian contraction via `contract_f12_cart` and applies one sph transform. For derivative variants:

1. After the primitive loop accumulates `cart_buf`, add a branch on `variant.ncomp`:
   - If ncomp == 1 (base variant): current behavior unchanged
   - If ncomp > 1 (derivative variant): call the appropriate gout function to produce `ncomp * nfi * nfj * nfk * nfl` values in the gout buffer

2. IMPORTANT: The gout functions operate on the G tensor BEFORE contraction and sph transform. So the flow must change to:
   - For derivative variants, the per-primitive contraction should NOT use `contract_f12_cart`. Instead, accumulate the raw G tensor contributions, then apply the gout function, then contract.
   - Actually re-reading the libcint code more carefully: `CINTgout2e_int2e_ip1` operates per-primitive (inside the primitive loop), computing `nf * ncomp` values from each primitive's G tensor. These are accumulated across primitives. Then `c2s_sph_2e1` is called ncomp times.
   - So the correct approach: inside the primitive loop, call fill_g_tensor_f12 to get the per-primitive G tensor, then apply the variant-specific gout function to produce ncomp * nf Cartesian values, then accumulate with contraction coefficients into a `gout_buf` of size `n_ctr_total * ncomp * nf`.

3. Revised primitive loop for ncomp > 1:
   ```
   let nf = ncart(li_u8) * ncart(lj_u8) * ncart(lk_u8) * ncart(ll_u8);
   let mut gout_contracted = vec![0.0f64; ncomp * nf];
   
   for pi, pj, pk, pl in primitive_loops:
       g = fill_g_tensor_f12(...)
       prim_gout = variant_gout_fn(&g, &shape, li, lj, lk, ll, ai, aj, ak, al)
       // prim_gout has ncomp * nf elements
       for ci, cj, ck, cl in contraction_loops:
           weight = coeff_i * coeff_j * coeff_k * coeff_l
           for idx in 0..ncomp * nf:
               gout_contracted[idx] += weight * prim_gout[idx]
   ```
   Wait - libcint separates contraction from gout. The contraction in `CINTf_2e_loop` produces `nc * n_comp` values where `nc = nf * i_ctr * j_ctr * k_ctr * l_ctr`. Then c2s is called per contraction block per component.

   For simplicity in cintx (STO-3G = 1 contraction per shell), the contraction is trivially a single accumulation. The implementation should:
   - Compute gout (ncomp * nf values) per primitive
   - Accumulate with contraction weights into `contracted_gout` (ncomp * nf values)
   - After primitive loops, apply cart_to_sph_2e separately to each of the ncomp blocks

4. After primitive loop, apply sph transform per component:
   ```
   for comp in 0..ncomp:
       let cart_slice = &contracted_gout[comp * nf .. (comp+1) * nf];
       let sph = cart_to_sph_2e(cart_slice, li_u8, lj_u8, lk_u8, ll_u8);
       // Note: transform uses base angular momenta li,lj,lk,ll (not ceil)
       let sph_size = nsi * nsj * nsk * nsl;
       staging[comp * sph_size .. (comp+1) * sph_size].copy_from_slice(&sph[..sph_size]);
   ```

5. The `nf` for gout indexing uses the BASE angular momenta li,lj,lk,ll (NOT the ceil values). The ceil values are only used for the G tensor shape (VRR/HRR needs extra headroom). The gout function iterates over `nf = ncart(li) * ncart(lj) * ncart(lk) * ncart(ll)` basis function tuples, using indices that reach up to li (base) for contraction, but the nabla operator reaches into the extra headroom at li+1 (or li+2 for ipip1).

6. For the base variant (ncomp=1), keep the existing `contract_f12_cart` + single `cart_to_sph_2e` path unchanged to avoid regressions.
