    **Structure:**

    1. **`launch_f12` dispatcher:** Reads `plan.descriptor.entry.operator_name` to route:
       - Names starting with `"stg"` route to STG entry points
       - Names starting with `"yp"` route to YP entry points
       - Extract variant suffix: `""` (base), `"_ip1"`, `"_ipip1"`, `"_ipvip1"`, `"_ip1ip2"`
       - Extract `f12_zeta` from `plan.operator_env_params.f12_zeta.unwrap()` (validator already confirmed non-zero per D-02)

    2. **Derivative variant increments (from `cint2e_f12.c` ng arrays):**
       ```rust
       struct F12Variant {
           i_inc: usize,  // added to li
           j_inc: usize,  // added to lj
           k_inc: usize,  // added to lk
           l_inc: usize,  // added to ll
           ncomp: usize,  // output component count
       }
       ```
       | Variant | i_inc | j_inc | k_inc | l_inc | ncomp |
       |---------|-------|-------|-------|-------|-------|
       | base    | 0     | 0     | 0     | 0     | 1     |
       | ip1     | 1     | 0     | 0     | 0     | 3     |
       | ipip1   | 2     | 0     | 0     | 0     | 9     |
       | ipvip1  | 1     | 1     | 0     | 0     | 9     |
       | ip1ip2  | 1     | 0     | 1     | 0     | 9     |

    3. **Shared F12 kernel core (called by all 10 entry points):**

       ```rust
       fn f12_kernel_core(
           backend: &ResolvedBackend,
           plan: &ExecutionPlan<'_>,
           specialization: &SpecializationKey,
           staging: &mut [f64],
           zeta: f64,
           variant: &F12Variant,
           is_stg: bool,  // true=STG, false=YP
       ) -> Result<ExecutionStats, cintxRsError>
       ```

       The core function follows the same structure as `launch_two_electron` in `two_electron.rs`:
       a. Extract shell angular momenta li, lj, lk, ll from plan
       b. Apply derivative increments: `li_ceil = li + variant.i_inc`, etc.
       c. Call `build_2e_shape(li_ceil, lj_ceil, lk_ceil, ll_ceil)` -- reuse the existing function from two_electron.rs (may need to make it `pub(crate)` or duplicate/refactor)
       d. Loop over primitive quartets (same as two_electron.rs)
       e. **ROOT COMPUTATION (F12-specific):**
          - Compute `a0 = aij * akl / (aij + akl)` (Gaussian product exponent)
          - Compute `ta = a0 * rij_kl_dist_sq` (reduced exponent times distance squared)
          - Compute `ua = 0.25 * zeta * zeta / a0`
          - Call `stg_roots_host(nroots, ta, ua)` to get `(roots, weights)`
       f. **WEIGHT POST-PROCESSING (STG vs YP divergence per D-04):**
          For STG (from g2e_f12.c lines 290-296):
          ```rust
          let ua2 = 2.0 * ua / zeta;
          for irys in 0..nroots {
              weights[irys] *= (1.0 - roots[irys]) * ua2;
              roots[irys] = roots[irys] / (1.0 - roots[irys]);
          }
          ```
          For YP (from g2e_f12.c lines 197-200):
          ```rust
          for irys in 0..nroots {
              weights[irys] *= roots[irys];
              roots[irys] = roots[irys] / (1.0 - roots[irys]);
          }
          ```
       g. After root post-processing, the transformed roots are used in the VRR 2D fill exactly as in `two_electron.rs` (same `b00`, `b10`, `b01`, `c00x/y/z`, `c0px/y/z` formulas)
       h. Apply HRR 4D transfer, Cartesian contraction, and cart_to_sph_2e -- identical to two_electron.rs
       i. For derivative variants with ncomp > 1, apply the appropriate `gout` indexing: ip1 produces 3 components (x,y,z gradients), ipip1/ipvip1/ip1ip2 produce 9 components. The `gout` pattern mirrors the existing `CINTgout2e_int2e_ip1` etc. from libcint, which indexes the g-tensor with derivative-shifted indices.

    **CRITICAL correctness notes:**
    - The VRR/HRR pipeline after root post-processing is IDENTICAL to plain 2e. Do not duplicate the logic; factor out common functions from `two_electron.rs` or call them directly. Make `build_2e_shape`, `cart_comps`, `common_fac_sp`, and the VRR/HRR fill functions `pub(crate)` if needed.
    - nroots is computed from the derivative-adjusted angular momenta: `nroots = (li_ceil + lj_ceil + lk_ceil + ll_ceil) / 2 + 1` (Pitfall 6)
    - The `common_factor` includes `fac` and `fac_sp` per the two_electron pattern
    - For STO-3G (s/p only), derivative variants ip1 with li_ceil=1 still need correct nroots

    4. **10 individual entry point functions (per D-03):**

       fn launch_stg_ip1ip2(...) { f12_kernel_core(..., &F12_IP1IP2, true) }
       fn launch_yp_base(...) { f12_kernel_core(..., &F12_BASE, false) }
       fn launch_yp_ip1(...) { f12_kernel_core(..., &F12_IP1, false) }
       fn launch_yp_ipip1(...) { f12_kernel_core(..., &F12_IPIP1, false) }
       fn launch_yp_ipvip1(...) { f12_kernel_core(..., &F12_IPVIP1, false) }
       fn launch_yp_ip1ip2(...) { f12_kernel_core(..., &F12_IP1IP2, false) }
